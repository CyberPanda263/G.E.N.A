Усі важливі зміни у проєкті **G.E.N.A. (GitOps & Engineering Network Assistant)** документуються в цьому файлі. Формат базується на [Keep a Changelog](https://keepachangelog.com/en/1.0.0/).

## [6.0.0] — 2026-05-30
### 🚀 Додано
* **Нативна інтеграція MinIO:** Розгортання S3-сумісного сховища через нативний Deployment & Service (замість legacy Helm-чартів) та автоматичне створення бакетів за допомогою `mc` клієнта.
* **Tekton SSL Fix:** Уніфікована конфігурація SSL/Ingress для доступу до Tekton Dashboard та Webhook Listener.
### 🛠 Покращення
* Оптимізована маршрутизація Velero до локального сервісу `minio-service.minio.svc.cluster.local:9000`.

## [5.2.0] — 2026-05-28
### 🛠 Покращення
* **Anti-Race Condition Bugfixes:** Покращено стабільність завантаження Tekton Webhooks для запобігання помилкам при ініціалізації тригерів.

## [5.1.0] — 2026-05-25
### 🚀 Додано
* **Auto-Env:** Інтегровано автоматичне завантаження конфігурації з файлу `.env` при його наявності, що спрощує процес розгортання.

## [5.0.0] — 2026-05-24
### 🚀 Додано
* **In-Cluster CI/CD:** Додано стек Tekton (Pipelines, Triggers, Dashboard) для автоматизації збірки всередині кластера.
* **Local Registry:** Інтегровано локальний Docker Registry (порт 5000) для зберігання зібраних образів.
* **Hybrid Backups:** Впроваджено інтеграцію Velero та Longhorn як опції для резервного копіювання.


## [4.0.0] — 2026-05-23
### 🚀 Додано
* **Нова архітектура моніторингу:** Повна інтеграція зв'язки Prometheus + Loki для збору метрик та логів в одному місці (Grafana).
* **Signal Alerting Adapter:** Створено кастомний Python-адаптер для інтеграції `Alertmanager` з месенджером Signal через офіційну двофакторну авторизацію (QR-код у терміналі).
* **Enterprise Hardening (Безпека):** Автоматичне налаштування брандмауера UFW (Default Deny), оптимізація параметрів ядра `sysctl` для Kubernetes та встановлення Fail2Ban проти brute-force атак на SSH.
* **Інтерактивне меню встановлення:** Додано вибір режимів Let's Encrypt (Staging/Production) та типів розгортання Portainer (Agent/Server).

### ⚙️ Змінено
* **Міграція на K3s:** Повністю оновлено базовий рушій кластера на легковажний дистрибутив K3s версії `v1.31.6+k3s1` замість "важкого" ванільного Kubernetes.
* **GitOps за замовчуванням:** Панель ArgoCD тепер розгортається автоматично з увімкненим замовчуванням механізмом `selfHeal` та `prune`.
* **SSL через DNS-01:** Cert-Manager переведено на автоматичну перевірку челенджів через Cloudflare API Токени, що дозволяє випускати сертифікати для закритих/локальних хостів.

### 🔒 Безпека
* Вимкнено автентифікацію по паролях для SSH (`PasswordAuthentication no`).
* Заборонено прямий логін для користувача `root` за паролем.
