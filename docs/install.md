## 📋 Попередні вимоги (Prerequisites)

Перед початком переконайтеся, що ваш цільовий вузол відповідає наступним вимогам:
* **ОС:** Чиста система Debian 12 або Ubuntu 22.04+ LTS.
* **Мережа:** Наявність публічної IP-адреси.
* **DNS:** Домени для сервісів заздалегідь делеговані у ваш акаунт **Cloudflare**.

## 🚀 Покрокове розгортання

### Крок 1. Підготовка змінних середовища

Перед запуском інсталяційного скрипта необхідно експортувати обов'язкові змінні оточення. Вони використовуються для автоматичного налаштування сертифікатів Let's Encrypt, інтеграції з API Cloudflare, авторизації у Signal та конфігурації доменів сервісів.

Виконайте наступні команди у вашому терміналі або збережіть їх у файл `.env` та виконайте :

```bash
# =========================================================
# ОБОВ'ЯЗКОВІ ЗМІННІ СЕРЕДОВИЩА (REQUIRED ENV)
# =========================================================
USER_EMAIL="your-email@example.com"

# Конфігурація Signal
SIGNAL_NUMBER="+380XXXXXXXXX"
SIGNAL_RECIPIENTS="+380YYYYYYYYY,+380ZZZZZZZZZ"
WEBHOOK_TOKEN="super-secret-auth-token-2026"

# Мережеві домени (FQDN)
GRAFANA_HOST="grafana.yourdomain.com"
ARGOCD_HOST="argocd.yourdomain.com"
TEKTON_HOST="tekton.yourdomain.com"

# Паролі та Токени
GRAFANA_ADMIN_PASSWORD="КастомнийНадійнийПароль123!"
CF_API_TOKEN="your_cloudflare_dns_zone_api_token_here"

# =========================================================
# ОПЦІОНАЛЬНІ ЗМІННІ (ЗА ЗАМОВЧУВАННЯМ)
# =========================================================
PORTAINER_HOST="portainer.yourdomain.com"
MINIO_HOST="minio.yourdomain.com"
MINIO_ROOT_USER="admin"
MINIO_ROOT_PASSWORD="MinioSuperSecretPassword!"
```

> ⚠️ **ВАЖЛИВО:** Перевірте правильність заповнення `CF_API_TOKEN`. Без коректного токена Cert-Manager не зможе автоматично випустити SSL-сертифікати через DNS-01 challenge, і доступ до веб-інтерфейсів по HTTPS буде заблоковано.

### Крок 2. Запуск інсталяційного скрипта

Завантажте скрипт на сервер, перейдіть у режим суперкористувача (`root`) для конфігурації системних параметрів `sysctl`, налаштування брандмауера UFW та встановлення пакетів, після чого запустіть встановлення:

```bash
# Перехід у режим root
sudo -i

# Створіть директорію в якій буде розгорнута платформа і перенесіть туди файл gena_deploy.sh та файл .env
mkdir -p /opt/gena-platform
cd /opt/gena-platform

# Надайте файлу права на виконання та запустіть (переконайтеся, що змінні експортовані)
chmod +x gena_deploy.sh
./gena_deploy.sh
```

### Крок 3. Інтерактивна конфігурація

Під час виконання скрипт зупиниться та запропонує обрати режими роботи через зручне текстове меню:

1. **Режим Let's Encrypt (SSL):**
   * `1) Staging` *(За замовчуванням)* — Тестовий сервер ACME без лімітів на запити. Рекомендується для первинного запуску. Браузери показуватимуть попередження.
   * `2) Production` — Бойовий режим з лімітом від Let's Encrypt (до 5 ідентичних сертифікатів на тиждень). Використовуйте після успішного тесту в Staging.
2. **Тип встановлення Portainer:**
   * `1) Portainer Agent` *(За замовчуванням)* — Встановлює легковагий агент. Використовується для підключення цього кластера до вже існуючого центрального Portainer Server.
   * `2) Portainer Server` — Розгортає повноцінну незалежну панель управління кластером (потребує налаштованого домену `PORTAINER_HOST`).

---

### Крок 4. Авторизація Signal (Увага на екран!)

Для інтеграції системи моніторингу кластера з месенджером Signal скрипт використовує офіційну двофакторну лінковку пристрою:

1. Скрипт запустить фоновий короткоживучий Pod `signal-linker` у просторі імен `monitoring`.
2. На екран терміналу буде виведено графічний **QR-код** та його текстовий аналог.
3. **Ваші дії:**
   * Відкрийте додаток **Signal** на своєму смартфоні.
   * Перейдіть у **Налаштування** -> **Зв'язані пристрої** (Linked Devices).
   * Натисніть кнопку **Додати новий пристрій (+)** та відскануйте QR-код з екрана терміналу.
4. Після успішного сканування скрипт автоматично видалить тимчасовий Pod і продовжить розгортання платформи.

## 🔐 Доступи та перевірка успішності

Після завершення конфігурації всіх інфраструктурних шарів скрипт виконає фінальну валідацію статусів (`kubectl rollout status`) та виведе підсумкову таблицю.

### Адреси сервісів та облікові дані

| Компонент | Маршрут доступу (URL) | Стандартний користувач | Пароль доступу |
| :--- | :--- | :--- | :--- |
| **Cockpit** (Панель ОС) | `https://<IP_ВАШОГО_СЕРВЕРА>:9090` | Локальний користувач Linux | Пароль користувача Linux |
| **Grafana** (Метрики/Логи) | `https://<Ваш_GRAFANA_HOST>` | `admin` | Значення з `$GRAFANA_ADMIN_PASSWORD` |
| **ArgoCD** (GitOps-панель) | `https://<Ваш_ARGOCD_HOST>` | `admin` | Згенерується автоматично (буде в консолі) |
| **Portainer** (якщо Server) | `https://<Ваш_PORTAINER_HOST>` | `admin` | Встановлюється індивідуально при першому вході |

> 💡 **Примітка щодо Portainer Agent:** Якщо обрано режим агента, скрипт згенерує унікальний рядок підключення вигляду `tcp://<Node_IP>:<Node_Port>`. Додайте його як нове оточення (Environment) у вашій головній консолі Portainer Server.

## 🛡️ Мережева безпека (Enterprise Hardening)

Платформа розгортається у максимально захищеному системному режимі:

* **SSH Hardening:** Автоматично вимикається автентифікація за паролями (`PasswordAuthentication no`) та забороняється прямий вхід для `root` за паролем. **Обов'язково додайте свій публічний SSH-ключ до файлу `~/.ssh/authorized_keys` ДО запуску інсталяції.**
* **Брандмауер UFW:** Всі вхідні з'єднання заблоковані за замовчуванням (**Default Deny**), крім явно дозволених портів:
  * `22/tcp` — SSH віддалений доступ.
  * `80/tcp` — HTTP веб-трафік (для редіректу на HTTPS та верифікації сертифікатів).
  * `443/tcp` — HTTPS захищений веб-трафік Ingress NGINX.
  * `6443/tcp` — Kubernetes API Server (для віддаленого керування через `kubectl`).
  * `9090/tcp` — Cockpit веб-інтерфейс адміністрування ОС.
* **Внутрішня мережа:** Повністю дозволено внутрішній трафік віртуальних інтерфейсів Kubernetes Flannel CNI (`cni0`, `flannel.1`).
* **Захист від Brute-Force:** Встановлюється та активується сервіс **Fail2Ban** для захисту порту SSH. Час блокування зловмисників становить 1 годину після 5 невдалих спроб автентифікації поспіль.
